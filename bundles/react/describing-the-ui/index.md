# Describing the UI

- [Describing the UI](describing-the-ui.md) - Section landing page for creating, customizing, and conditionally displaying React components.
- [Your First Component](your-first-component.md) - Components are reusable UI building blocks: JavaScript functions, named with a capital letter, that return JSX markup.
- [Importing and Exporting Components](importing-and-exporting-components.md) - Split components across files with default and named imports and exports.
- [Writing Markup with JSX](writing-markup-with-jsx.md) - JSX is the HTML-like syntax for markup in JavaScript, with three rules: one root element, close all tags, camelCase attributes.
- [JavaScript in JSX with Curly Braces](javascript-in-jsx-with-curly-braces.md) - Curly braces open a window to JavaScript inside JSX for variables, calls, and objects.
- [Passing Props to a Component](passing-props-to-a-component.md) - Configure components with props, any JavaScript value, including defaults, spread, and children.
- [Conditional Rendering](conditional-rendering.md) - Render different JSX depending on conditions using if, the logical AND, and the ternary operator.
- [Rendering Lists](rendering-lists.md) - Render multiple components from a collection using map() and filter(), each with a stable key.
- [Keeping Components Pure](keeping-components-pure.md) - Write components as pure functions that return the same output for the same inputs to avoid bugs.
- [Understanding Your UI as a Tree](understanding-your-ui-as-a-tree.md) - Model your UI as a render tree and a module dependency tree to reason about performance and bundle size.
