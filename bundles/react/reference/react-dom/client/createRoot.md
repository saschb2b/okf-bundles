# Reference

```js
import { createRoot } from 'react-dom/client';

const domNode = document.getElementById('root');
const root = createRoot(domNode);
root.render(<App />);
```

Creates a React root for `domNode` and takes over managing the DOM inside it. Call `root.render` to display a component. For server-rendered HTML use [`hydrateRoot`](/reference/react-dom/client/hydrateRoot.md) instead.

## Parameters

- `domNode`: a DOM element. React creates a root for it and lets you call methods like `render`.
- optional `options`: an object:
  - optional `onCaughtError`: callback when React catches an error in an Error Boundary. Receives the `error` and an `errorInfo` object with a `componentStack`.
  - optional `onUncaughtError`: callback when an error is thrown and not caught by an Error Boundary. Receives the `error` and an `errorInfo` with a `componentStack`.
  - optional `onRecoverableError`: callback when React automatically recovers from an error. Receives the `error` and an `errorInfo` with a `componentStack`. Some errors include the original cause as `error.cause`.
  - optional `identifierPrefix`: a string prefix for IDs generated by [`useId`](/reference/react/hooks/useId.md). Useful to avoid conflicts across multiple roots on one page.

## Returns

An object with two methods: `render` and `unmount`.

# Root methods

## `root.render(reactNode)`

Displays a React node (usually JSX like `<App />`, but also a `createElement` result, string, number, `null`, or `undefined`) into the root. Returns `undefined`.

- The first call clears existing HTML inside the root before rendering.
- Calling it again on the same root updates the DOM by matching against the previous tree, reusing nodes where possible, like calling a state setter on the root.
- `root.render(...)` is not fully synchronous: code after it may run before that render's `useLayoutEffect` or `useEffect` fire. Wrap in [`flushSync`](/reference/react-dom/apis/flushSync.md) in the rare case effect timing matters.

## `root.unmount()`

Destroys the rendered tree, unmounts all components, and detaches React from the root DOM node, removing event handlers and state. Takes no parameters and returns `undefined`. Useful when other code (for example a jQuery panel) may remove the root's DOM node, so components can clean up. After unmounting you cannot call `render` again on that root, but you can create a new root for the same node.

# Usage

- Render an app fully built with React: create a single root and call `render` once at startup.
- Render a page partially built with React: call `createRoot` multiple times, one root per top-level React-managed piece, then `render` into each.
- Update a root component: calling `render` again preserves state when the tree structure matches, though usually you update state inside components instead.
- Error logging in production: pass `onCaughtError`, `onUncaughtError`, and `onRecoverableError` to build a custom error reporting system.

# Caveats

- Not supported for server-rendered apps. Use [`hydrateRoot`](/reference/react-dom/client/hydrateRoot.md).
- You usually have only one `createRoot` call, often done by your framework.
- To render JSX outside your component's DOM subtree (a modal or tooltip), use [`createPortal`](/reference/react-dom/apis/createPortal.md), not another root.

# Troubleshooting

- Nothing is displayed: you forgot to call `root.render`.
- "You passed a second argument to root.render": pass options to `createRoot`, not `root.render`.
- "Target container is not a DOM element": the argument is not a DOM node (for example `getElementById` returned `null` due to an ID typo or a script that runs before the node exists), or you wrote `createRoot(<App />)` instead of `createRoot(domNode)`.
- "Functions are not valid as a React child": you passed `App` or `createApp` instead of `<App />` or `createApp()` to `render`.
- Server-rendered HTML re-created from scratch: use `hydrateRoot` instead of `createRoot`.
