# react-dom (DOM rendering)

The `react-dom` package: rendering React to the browser DOM and to HTML on the server, plus the browser-specific components and APIs.

- [Components](components/index.md) - Built-in browser components and React's special handling of them: `<form>`, `<input>`, `<select>`, `<textarea>`, `<link>`, `<meta>`, `<script>`, `<style>`, `<title>`, and the common props shared by all DOM elements.
- [Hooks](hooks/index.md) - `useFormStatus`, for reading the status of a parent `<form>` submission.
- [APIs](apis/index.md) - `createPortal`, `flushSync`, and the resource preloading APIs (`preload`, `preinit`, `preconnect`, `prefetchDNS`, and the module variants).
- [Client APIs](client/index.md) - `createRoot` and `hydrateRoot`, for rendering in the browser (`react-dom/client`).
- [Server APIs](server/index.md) - `renderToPipeableStream`, `renderToReadableStream`, `renderToString`, and more, for rendering to HTML on the server (`react-dom/server`).
- [Static APIs](static/index.md) - `prerender` and `prerenderToNodeStream`, for static site generation (`react-dom/static`).
