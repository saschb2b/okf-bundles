# Reference

`prerenderToNodeStream` renders a React tree to static HTML using a Node.js Stream. It is a static site generation (SSG) API that waits for all data to load before resolving, including Suspense-fetched data.

```js
import { prerenderToNodeStream } from 'react-dom/static';

const { prelude, postponed } = await prerenderToNodeStream(reactNode, options?)
```

Runtime: Node.js Streams only. For Web Streams environments (Deno, modern edge runtimes), use [prerender](/reference/react-dom/static/prerender.md) instead.

## Parameters

- `reactNode`: The React node to render to HTML, for example `<App />`. It is expected to represent the entire document, so the component should render the `<html>` tag.
- `options` (optional): An object with static generation options.
  - `bootstrapScriptContent`: A string placed in an inline `<script>` tag.
  - `bootstrapScripts`: An array of string URLs for `<script>` tags to emit on the page. Use this to include the script that calls [hydrateRoot](/reference/react-dom/client/hydrateRoot.md). Omit it if you do not want to run React on the client.
  - `bootstrapModules`: Like `bootstrapScripts`, but emits `<script type="module">`.
  - `identifierPrefix`: A string prefix for IDs generated by [useId](/reference/react/hooks/useId.md). Must match the prefix passed to [hydrateRoot](/reference/react-dom/client/hydrateRoot.md).
  - `namespaceURI`: Root namespace URI. Defaults to HTML. Pass `'http://www.w3.org/2000/svg'` for SVG or `'http://www.w3.org/1998/Math/MathML'` for MathML.
  - `onError`: Callback fired on any server error, recoverable or not. Defaults to `console.error`. Can also adjust the status code before the shell is emitted.
  - `progressiveChunkSize`: The number of bytes in a chunk.
  - `signal`: An abort signal to abort prerendering and render the rest on the client.

## Returns

A Promise that resolves on success to an object with:

- `prelude`: A Node.js Stream of HTML. Send it in chunks or read it entirely into a string.
- `postponed`: A JSON-serializable, opaque object passable to [resumeToPipeableStream](/reference/react-dom/server/resumeToPipeableStream.md) if `prerenderToNodeStream` did not finish. `null` if `prelude` contains all content and no resume is necessary.

On failure, the Promise rejects. Use that to output a fallback shell.

# Usage

- Render to a response stream. The root component should return the entire document, and React injects the doctype and bootstrap scripts.

```js
app.use('/', async (request, response) => {
  const { prelude } = await prerenderToNodeStream(<App />, {
    bootstrapScripts: ['/main.js'],
  });
  response.setHeader('Content-Type', 'text/html');
  prelude.pipe(response);
});
```

- Render to a string by accumulating `data` events.

```js
let data = '';
prelude.on('data', chunk => { data += chunk; });
prelude.on('end', () => resolve(data));
prelude.on('error', reject);
```

- Wait for all data. Only Suspense-enabled sources activate Suspense: Suspense-enabled frameworks (Relay, Next.js), code lazy-loaded with [lazy](/reference/react/apis/lazy.md), and Promises read with [use](/reference/react/apis/use.md).
- Abort after a timeout with an `AbortController` passed as `signal`. Incomplete Suspense boundaries appear in the prelude in their fallback state. Continue later with [resumeToPipeableStream](/reference/react-dom/server/resumeToPipeableStream.md) or [resumeAndPrerenderToNodeStream](/reference/react-dom/static/resumeAndPrerenderToNodeStream.md) for partial prerendering.

# Caveats

- This API is specific to Node.js. For Web Streams, use [prerender](/reference/react-dom/static/prerender.md).
- `nonce` is not an available option when prerendering, since a nonce must be unique per request and would be insecure under CSP if baked into the prerender.
- Does not stream as content loads. It waits for the entire app to finish. To stream, use [renderToPipeableStream](/reference/react-dom/server/renderToPipeableStream.md).
