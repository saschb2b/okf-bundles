# Reference

`renderToPipeableStream` renders a React tree to a pipeable Node.js Stream, streaming HTML as content becomes ready.

```js
import { renderToPipeableStream } from 'react-dom/server';

const { pipe, abort } = renderToPipeableStream(reactNode, options?)
```

Runtime: Node.js Streams only. For Web Streams environments (Deno, modern edge runtimes), use [renderToReadableStream](/reference/react-dom/server/renderToReadableStream.md).

## Parameters

- `reactNode`: The React node to render to HTML, for example `<App />`. It is expected to represent the entire document, so the component should render the `<html>` tag.
- `options` (optional): An object with streaming options.
  - `bootstrapScriptContent`: A string placed in an inline `<script>` tag.
  - `bootstrapScripts`: An array of string URLs for `<script>` tags to emit on the page. Use this to include the script that calls [hydrateRoot](/reference/react-dom/client/hydrateRoot.md).
  - `bootstrapModules`: Like `bootstrapScripts`, but emits `<script type="module">`.
  - `identifierPrefix`: A string prefix React uses for IDs generated by [useId](/reference/react/hooks/useId.md). Must match the prefix passed to `hydrateRoot`.
  - `namespaceURI`: Root namespace URI. Defaults to HTML. Pass `'http://www.w3.org/2000/svg'` for SVG or `'http://www.w3.org/1998/Math/MathML'` for MathML.
  - `nonce`: A nonce string to allow scripts for `script-src` Content-Security-Policy.
  - `onAllReady`: Callback that fires when all rendering is complete (shell plus additional content). Use for crawlers and static generation instead of `onShellReady`.
  - `onError`: Callback that fires on any server error, recoverable or not. Use for logging crash reports.
  - `onShellReady`: Callback that fires right after the initial shell is rendered. Call `pipe` here to start streaming.
  - `onShellError`: Callback that fires if there was an error rendering the initial shell. No bytes have been emitted yet.
  - `progressiveChunkSize`: The number of bytes in a chunk.

## Returns

An object with two methods:

- `pipe`: Outputs the HTML into a provided Writable Node.js Stream. Call in `onShellReady` to enable streaming, or in `onAllReady` for crawlers and static generation.
- `abort`: Aborts server rendering and renders the rest on the client.

# Usage

- Stream HTML to a response. Call `pipe(response)` inside `onShellReady`, after setting the content type header.

```js
const { pipe } = renderToPipeableStream(<App />, {
  bootstrapScripts: ['/main.js'],
  onShellReady() {
    response.setHeader('content-type', 'text/html');
    pipe(response);
  }
});
```

- Stream more content as it loads with Suspense. The shell is everything outside all `<Suspense>` boundaries. React sends fallbacks first, then streams the resolved content.

```jsx
<Suspense fallback={<PostsGlimmer />}>
  <Posts />
</Suspense>
```

- Wait for all content before streaming, for crawlers and static generation. Call `pipe` in `onAllReady` instead of `onShellReady`.
- Abort after a timeout. Call `abort()` so React flushes remaining loading fallbacks and falls back to client rendering.

```js
setTimeout(() => abort(), 10000);
```

- Hydrate on the client. Call [hydrateRoot](/reference/react-dom/client/hydrateRoot.md) to make the server HTML interactive.

# Caveats

- This API is specific to Node.js. Use [renderToReadableStream](/reference/react-dom/server/renderToReadableStream.md) where Web Streams are available.
- Place `<Suspense>` boundaries so the shell is minimal but complete, since the shell determines the earliest loading state users see.
- Use the `onShellError` and `onError` callbacks to set an appropriate HTTP status code before bytes are emitted.
