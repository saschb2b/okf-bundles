# Reference

`renderToReadableStream` renders a React tree to a Readable Web Stream, streaming HTML as content becomes ready.

```js
import { renderToReadableStream } from 'react-dom/server';

const stream = await renderToReadableStream(reactNode, options?)
```

Runtime: Web Streams API (Deno, modern edge runtimes). For Node.js, use [renderToPipeableStream](/reference/react-dom/server/renderToPipeableStream.md) instead.

## Parameters

- `reactNode`: The React node to render to HTML, for example `<App />`. It is expected to represent the entire document, including the `<html>` tag.
- `options` (optional): An object with streaming options.
  - `bootstrapScriptContent`: A string placed in an inline `<script>` tag.
  - `bootstrapScripts`: An array of string URLs for `<script>` tags to emit on the page.
  - `bootstrapModules`: Like `bootstrapScripts`, but emits `<script type="module">`.
  - `identifierPrefix`: A string prefix React uses for IDs generated by [useId](/reference/react/hooks/useId.md). Must match the prefix passed to [hydrateRoot](/reference/react-dom/client/hydrateRoot.md).
  - `namespaceURI`: Root namespace URI. Defaults to HTML. Pass `'http://www.w3.org/2000/svg'` for SVG or `'http://www.w3.org/1998/Math/MathML'` for MathML.
  - `nonce`: A nonce string to allow scripts for `script-src` Content-Security-Policy.
  - `onError`: A callback fired on any server error, recoverable or not. Defaults to `console.error`. Use for logging crash reports and adjusting status codes.
  - `progressiveChunkSize`: The number of bytes in a chunk.
  - `signal`: An abort signal to abort server rendering and render the rest on the client.

## Returns

A Promise:

- If shell rendering succeeds, resolves to a Readable Web Stream.
- If shell rendering fails, the Promise is rejected. Catch it to output a fallback shell.

The returned stream has an additional property:

- `allReady`: A Promise that resolves when all rendering is complete (shell plus all additional content). `await stream.allReady` before returning a response to wait for the final HTML, useful for crawlers and static generation.

# Usage

- Render and return a streaming response. The root component should render the entire document.

```js
async function handler(request) {
  const stream = await renderToReadableStream(<App />, {
    bootstrapScripts: ['/main.js']
  });
  return new Response(stream, {
    headers: { 'content-type': 'text/html' },
  });
}
```

- Stream more content as it loads with Suspense. The shell (content outside `<Suspense>`) streams immediately, the rest follows as it resolves.

```jsx
<Suspense fallback={<PostsGlimmer />}>
  <Posts />
</Suspense>
```

- Handle errors inside the shell with try/catch around the `await`, returning a 500 fallback. Handle errors outside the shell with the `onError` callback.
- Wait for all content for crawlers. `await stream.allReady` before returning, at the cost of progressive loading.
- Abort after a timeout via an `AbortController` passed as `signal`.

```js
const controller = new AbortController();
setTimeout(() => controller.abort(), 10000);
const stream = await renderToReadableStream(<App />, {
  signal: controller.signal,
  bootstrapScripts: ['/main.js']
});
```

- Hydrate on the client with [hydrateRoot](/reference/react-dom/client/hydrateRoot.md).

# Caveats

- This API depends on Web Streams. For Node.js, use [renderToPipeableStream](/reference/react-dom/server/renderToPipeableStream.md).
- The Promise rejects only on shell errors. Errors outside the shell reach `onError` but do not reject, since the shell already streamed.
