# Reference

`renderToStaticMarkup` renders a non-interactive React tree to an HTML string. Use it for fully static content like emails or a simple static page generator.

```js
import { renderToStaticMarkup } from 'react-dom/server';

const html = renderToStaticMarkup(reactNode, options?)
```

Runtime: Server-side (Node.js or any backend). It also runs in the browser, but using it in client code is not recommended.

## Parameters

- `reactNode`: The React node to render to HTML, for example `<Page />`.
- `options` (optional): An object for server render.
  - `identifierPrefix`: A string prefix React uses for IDs generated by [useId](/reference/react/hooks/useId.md). Useful to avoid conflicts when using multiple roots on the same page.

## Returns

An HTML string.

# Usage

- Render a non-interactive tree to a string and send it as the server response.

```js
app.use('/', (request, response) => {
  const html = renderToStaticMarkup(<Page />);
  response.send(html);
});
```

# Caveats

- The output cannot be hydrated. For interactive apps, use [renderToString](/reference/react-dom/server/renderToString.md) on the server and [hydrateRoot](/reference/react-dom/client/hydrateRoot.md) on the client.
- Limited Suspense support. If a component suspends, `renderToStaticMarkup` immediately sends its fallback as HTML.
- It works in the browser, but to render a component to HTML on the client, render it into a DOM node instead.
