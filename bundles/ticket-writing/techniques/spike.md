# Definition

When a story is genuinely uncertain, invest a fixed, capped amount of effort to learn (prototype, read, test feasibility), then decide: estimate it, split it, or drop it. The investment is bounded up front, so the unknown cannot consume the sprint.

# Caution

Use spikes only for extreme or excessive uncertainty. Overusing them extends time to value; carry ordinary uncertainty instead of spiking it.
